# ขั้นตอนที่ 3.8 บันทึกผลการเปรียบเทียบ

## ตารางสรุปผลการเปรียบเทียบระหว่าง REST และ GraphQL

| หัวข้อเปรียบเทียบ                                        | REST (`/students/:id/full`)                                                  | GraphQL                                                         |
| :------------------------------------------------------- | :--------------------------------------------------------------------------- | :-------------------------------------------------------------- |
| **จำนวนคำขอที่ใช้เพื่อดูชื่อและรายวิชาของนักศึกษา 1 คน** | **1 คำขอ** แต่ได้รับข้อมูลทั้งหมดของนักศึกษา แม้จะไม่ต้องการทุกฟิลด์         | **1 คำขอ** และเลือกเฉพาะฟิลด์ที่ต้องการได้                      |
| **จำนวนคำขอที่ใช้เพื่อดูชื่อและรายวิชาของนักศึกษาทุกคน** | อาจต้องใช้ **มากกว่า 1 คำขอ** หากข้อมูลรายวิชาแยกอยู่คนละ Endpoint           | **1 คำขอ** สามารถดึงข้อมูลนักศึกษาพร้อมรายวิชาได้ทั้งหมด        |
| **การเกิด Over-fetching (ได้รับข้อมูลเกินความจำเป็น)**   | **มี** เช่น `email`, `phone`, `courseIds`, `id` และข้อมูลอื่นที่ไม่ได้ใช้งาน | **ไม่มี** เพราะเลือกเฉพาะฟิลด์ที่ต้องการ                        |
| **การเลือกฟิลด์ (Field Selection)**                      | ไม่สามารถเลือกฟิลด์เองได้ ต้องรับข้อมูลตามที่ API กำหนด                      | สามารถเลือกเฉพาะฟิลด์ที่ต้องการได้                              |
| **ความยืดหยุ่นในการดึงข้อมูล**                           | ค่อนข้างจำกัด หากต้องการข้อมูลรูปแบบใหม่มักต้องสร้าง Endpoint เพิ่ม          | มีความยืดหยุ่นสูง สามารถปรับ Query ได้ตามต้องการ                |
| **ปัญหา Under-fetching (ข้อมูลไม่ครบ)**                  | อาจเกิดขึ้น หากข้อมูลอยู่หลาย Endpoint จำเป็นต้องเรียกหลายครั้ง              | ไม่มี เพราะสามารถดึงข้อมูลที่เกี่ยวข้องทั้งหมดได้ใน Query เดียว |
| **ความเหมาะสมในการใช้งาน**                               | เหมาะกับระบบที่ไม่ซับซ้อน ใช้งานง่าย และเป็นมาตรฐานที่นิยม                   | เหมาะกับระบบที่มีข้อมูลเชื่อมโยงหลายส่วน และต้องการลดจำนวนคำขอ  |
